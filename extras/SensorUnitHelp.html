Help Document for KootNet Sensor Units

KootNet Sensors requires a Raspberry Pi to utilize Sensors but can be installed on Ubuntu or Debian in
order to use the “Sensor Control” web interface for managing up to 20 other remote Kootnet Sensors.

To make your own sensor, find the instructions on the Programs Website.
** Note: Additional hardware sensors for the Raspberry Pi are optional **


Manage the Sensor using a Terminal or SSH connection

The installer creates the following shortcuts in your system programs menu.

“Terminal Config - Kootnet Sensors” – Used to adjusting configurations, run system commands, test the sensors and change the username and password for logging into the Web Portal.

It can be accessed directly by running the following command in a terminal.

sudo /home/kootnet_data/env/bin/python /opt/kootnet-sensors/start_cli_edit_config.py

“Sensor Tests - Kootnet Sensors” – Gui application to test a sensors functionality.

“Web Portal - Kootnet Sensors” – Shortcut to open the local HTTPS web server in your default browser.

** Note: It is recommended to use the Web Portal for configuring the sensor, due to its ease of use **


Manage the Sensor using the built in HTTPS Web Portal
The sensor’s web portal allows you to interact with the local sensor, manage up to 20 remote sensors and configure it’s settings. You can access the portal from any web browser (Chrome recommended) by going to https://IP-Address:10065 on the local network. Replace “IP-Address” with the sensor’s actual IP. If you are accessing the web portal from the sensor itself, there is a shortcut in the accessories menu or you can go to https://localhost:10065 in the local web browser.


Locations
KootNet sensor program – /opt/kootnet-sensors/
SQLite3 Database (Holds all recorded data) - /home/kootnet_data/SensorRecordingDatabase.sqlite
Configuration Files - /etc/kootnet/

  • installed_sensors.conf – Supported & enabled sensors

  • main_config.conf – General configuration

  • trigger_variances.conf – Trigger Variance configuration


SSL Files for HTTPS - /opt/kootnet-sensors/http_server/ssl_files/

  • kootnet_default.key – Private Key

  • kootnet_default.csr – Certificate Signing Request

  • kootnet_default.crt – Certificate


Log Files - /home/kootnet_data/logs/

  • Primary_log.txt – Program logs

  • Network_log.txt – Network related logs

  • Sensors_log.txt – Sensor hardware logs


Clean Upgrade Service Scripts

These 2x Upgrade scripts uninstall the systemd service, deletes Kootnet Sensors program & shortcuts then finally re-installs the program by the chosen method. The configuration and data files remain untouched. ** Note: If the script is unable to access the location it is installing from (The HTTP Server in most cases), you will have to manually run the regular installer on the local sensor **

HTTPS Clean Upgrade

sudo bash /home/kootnet_data/scripts/clean_upgrade_http.sh


Uninstall Sensor Software
Sensor data, configurations & logs are left on the system.

sudo apt-get remove kootnet-sensors
