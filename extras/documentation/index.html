Introduction #back to top

Kootnet Sensors is a program used to take sensor readings from multiple sensors and record them to an SQLite3 database and or upload them to one or more Online 3rd party service like Open Sense Map. Add one or more of the 25+ supported hardware sensors to record and view readings of your choice. Sensor types include heat, pressure, altitude, humidity, distance, gas, particulate matter 1/2.5/4/10, light, color, ultra violet, acceleration, electromagnetic fields and angular velocity (gyroscope).

KootNet Sensors requires a Raspberry Pi to utilize Sensors but can be installed on Ubuntu or Debian in order to use the “Sensor Control” web interface for managing up to 20 other remote Kootnet Sensors as well as uploading other sensor databases in order to utilize more powerful computers in creating graphs.

Raspberry Pi

Note: Sensor names starting with a * are untested.

Installation & Removal #back to top

Requirements

Operating System : Ubuntu, Debian, Raspbian/Raspberry Pi OS
Other Requirements : Internet Access
Note : Most installs finish within a few minutes but the Raspberry Pi Zero can take up to 30 minutes
Note : You must use the 32bit Raspbian/Raspberry Pi OS for proper Hardware Sensor functionality

Installation using a GUI Desktop

Installation from a supported GUI based Operating system is fairly straight forward. Click one of the following links to download Kootnet sensors, then double click the downloaded file and look for an install button to start the install process.

Current Stable Release
Latest Developmental Release ** Warning: Developmental Releases can be unstable **

Once the install has completed, you can find shortcuts to the various program functions in your operating systems menu. In Raspbian / Raspberry Pi OS, you can find the program shortcuts under "Accessories".

Installation using a Terminal

To install using a terminal, enter the following command, hit enter and wait for it to finish.

wget -O KootnetSensors.deb http://kootenay-networks.com/installers/KootnetSensors.deb && sudo apt-get update && sudo apt-get -y install ./KootnetSensors.deb

If the installation completes successfully, you'll be given the default Username/Password & URL to access the sensor.

Removal using the Terminal

Run the following Terminal command to uninstall Kootnet Sensors.

sudo apt-get remove kootnet-sensors

Note: The Sensor's Databases, Configurations & Python Virtual Environment will be kept

Web Portal Basics #back to top

General Overview

Kootnet Sensors has a built in HTTPS server (Web Portal) to monitor, manage and operate the Sensor. There is also a 'Sensor Control' section in the Web Portal for managing one or more remote sensors. Assuming the sensor's IP is 192.168.10.11, you can access the sensor at https://192.168.10.11:10065 from any device with a web browser on the same network.

The following sections will go into more details regarding definitions and functionality.
Please note, Internet is required for viewing pictures in the following sections.

Default Web Portal Login
User: Kootnet
Password: sensors

Recommended: Change the default Web Portal username and password after install.
Note: Find shortcuts to edit configurations and access the Web Portal in the operating systems main menu.

System Information

The System Information page has all the information you need to get a quick overview of the Sensor's status.

Operating System: The Underlying Operating System the Sensor Software is running on top of
Sensor ID: Used to uniquely identify the sensor in 'Sensor Check-Ins'
Running Version: Version of Kootnet Sensors running locally
Newest Std Version: The latest available Standard (Stable) version of Kootnet Sensors
Newest Dev Version: The latest available Developmental (Unstable) version of Kootnet Sensors
Last Updated: Last date & time the local Kootnet Sensor software was updated
Sensor Date & Time: Date, time & timezone on the local Sensor
Sensor Uptime: Length of time the local sensor has been online without a reboot or shutdown
Sensor Reboots: Amount of times the Sensor has rebooted (According to uptime recordings in the database)
System Temperature: Current Temperature of the System's CPU
RAM Usage: % of RAM in use on the local Sensor
Drive Usage: Current % of disk space used on the root drive ('/')

Installed Sensors: Currently Installed Sensors (Most sensors will disable them selves if initialization fails)

Program Status: Status of the different aspects of Kootnet Sensors. Availible Status's are Disabled, Enabled

Sensor Readings

The Sensor Reading's Page gives a live snapshot of each sensor's readings. If the sensor is missing or not enabled, it will show "NoSensor". Most sensor names can be clicked to take you to the relevant Wikipedia page.

Database

The main Index page for Kootnet Sensors is the Database Information Page. On this page you can download the SQLite3 database in RAW or Zip format. You can also upload a RAW SQLite3 database to replace the existing one. This is meant to allow the creation of graphs on faster PCs and the transfer of old data to new sensors.

Note: The SQLite3 database holds all sensor recordings as well as Notes.
Note: The Sensor Checkin Database is a separate SQLite3 database that currently cannot be downloaded.

Notes

The Notes page is where you can add, modify and remove notes in the SQL database. There is a "Date/Time" function under the Insert menu meant to be used to add a standard date stamp of when you are working on your notes but also to identify exactly when you find an interesting data point in the sensor recordings. This way others can create a graph to locate data the note is refering to.

Plotly Graphing

Advanced

The Advanced page is used to execute functions that are more System related, as opposed to sensor related. Find links to upgrade Kootnet Sensors, the OS & Python Modules. You can also view RAW configurations, Python Module Versions & Sensor Checkin Statistics. Power, SQL and Login update functions are located here as well.

Logs

The Logs are broken into 3 tabs, Primary, Network and Sensors.
The Primary log holds general program operations information.
The Network Log holds information regarding network commands and operations.
The Sensors Log holds information regarding hardware sensor operations.

Terminal Configurations

The Terminal Configuration Tool allows you to control some aspects of the sensor using the sensor's local Terminal. To run the Terminal Configuration Tool, open a terminal and run the following command. You will be presented with a number of commands. 

sudo /home/kootnet_data/env/bin/python /opt/kootnet-sensors/start_cli_edit_config.py

View/Edit Configurations: Opens the Primary & Installed Sensors configurations in nano
Change Web Login Credentials: Reset's the Web Portal Login to values of your choice
Update Python Modules: Upgrades each Python module, one after the other, then restarts the application
Upgrade Kootnet Sensors
(Standard HTTP): Upgrades Kootnet Sensors to the latest Stable Version
(Development HTTP): Upgrades Kootnet Sensors to the latest Unstable Version
(Clean HTTP): Removes all program files minus databases, logs and configurations then re-installs the latest Stable version of Kootnet Sensors
Create New Self Signed SSL Certificate: Removes old Web SSL Certificates, Creates new ones & restarts the software
Enable & Start KootnetSensors: Enables & Starts Kootnet Sensor's systemd service
Disable & Stop KootnetSensors: Disables & Stops Kootnet Sensor's systemd service
Restart KootnetSensors Service: Restarts Kootnet Sensor's systemd service
Run Sensor Local Tests: Runs basic Sensor Tests and returns the results
Exit: Exit's the Terminal Configuration Tool

Installation & Upgrade Problems

Installation issues can occur for various reasons. Here are a few possible causes and fixes.

  • 1. Internet Connection Issues
    •      Errors may include: Timeout messages, server unresponsive, failed to satisfy dependencies
    •      - Find a more stable Internet connection
  • 2. Unresponsive Servers hosting required dependencies (Such as the Raspberry Pi Repositories)
    •      Errors may include: Timeout messages, server unresponsive, failed to satisfy dependencies
    •      - Wait 5 or 10 minutes and try again OR change your default apt-get repository

Kootnet Sensors Start-up Problems

  • 1. Invalid Primary or Trigger Configuration
    •      Errors may include: ???
    •      - Remove the Primary Configuration file and restart the sensor
    •      - Review the Trigger Configuration before re-enabling trigger recording
    sudo rm -f /etc/kootnet/main_config.conf && sudo reboot
  • 2. Bad Hardware (Such as a corrupt SD Card)
    •      Errors may include: Missing dependencies in the Kootnet Sensor logs
    •      - Ensure the Sensor is Online, then run "Upgrade Kootnet Sensors (Clean HTTP)" from the *Terminal Configurations Tool*
  • 3. File Corruption
    •      Errors may include: Missing dependencies in the Kootnet Sensor logs
    •      - Ensure the Sensor is Online, then run "Upgrade Kootnet Sensors (Clean HTTP)" from the *Terminal Configurations Tool*
